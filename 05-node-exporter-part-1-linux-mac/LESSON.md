# **Understanding Node Exporter in Prometheus**

Now that Prometheus is installed, the next step is collecting system metrics using one of the most important exporters in the Prometheus ecosystem:

# **Node Exporter**



---

# **1. What Is Node Exporter?**

Node Exporter is an official Prometheus exporter used to collect metrics from Unix-based operating systems.

Despite the name:

> Node Exporter has nothing to do with Node.js.

In Prometheus terminology:

* A **node** means a machine/server
* Usually a Unix/Linux-based system

Examples include:

* Ubuntu
* Debian
* CentOS
* Red Hat Enterprise Linux
* Amazon Linux

---

# **2. Why It Is Called “Node” Exporter**

In infrastructure and monitoring terminology:

```text id="9h6h1m"
Node = Server / Host / Machine
```

So Node Exporter simply means:

```text id="lcmg5r"
Exporter for collecting machine-level metrics
```

---

# **3. Node Exporter Is an Official Prometheus Project**

Node Exporter is maintained as part of the Prometheus ecosystem.

That means:

* It follows Prometheus standards
* It is officially supported
* It integrates directly with Prometheus scraping

---

# **4. Official vs Community Exporters**

Not all exporters are built by the Prometheus team.

## Official Exporters

Maintained by the Prometheus project.

Examples:

* Node Exporter
* Blackbox Exporter
* Pushgateway

---

## Community/Third-Party Exporters

Created by:

* Companies
* Open-source contributors
* Vendors
* Independent developers

Examples:

* CloudWatch exporters
* Database exporters
* Application-specific exporters

---

# **5. What Metrics Can Node Exporter Collect?**

Node Exporter exposes many operating system metrics.

Common examples include:

| Metric Category | Examples                     |
| --------------- | ---------------------------- |
| CPU             | CPU usage, load average      |
| Memory          | RAM usage, swap              |
| Disk            | Disk usage, filesystem stats |
| Network         | Network traffic, packets     |
| System          | Uptime, boot time            |
| Processes       | Running process counts       |

---

# **6. Typical Node Exporter Architecture**

```text id="z73o1f"
Linux Server → Node Exporter → Prometheus
```

---

# **7. How Prometheus Uses Node Exporter**

The workflow is:

1. Node Exporter runs on the Linux server
2. It exposes metrics on an HTTP endpoint
3. Prometheus scrapes the endpoint periodically
4. Metrics are stored inside Prometheus

---

# **8. Default Metrics Endpoint**

By default, Node Exporter exposes metrics at:

```text id="vh9k6i"
http://<SERVER_IP>:9100/metrics
```

Default port:

```text id="go5lbo"
9100
```

---

# **9. Example Metrics Exposed**

Example Prometheus metrics:

```text id="d8qjmf"
node_cpu_seconds_total
node_memory_MemAvailable_bytes
node_filesystem_size_bytes
node_network_receive_bytes_total
```

These metrics are automatically generated by Node Exporter.

---

# **10. Pull-Based Monitoring**

Prometheus does not receive pushed metrics from Node Exporter.

Instead:

```text id="1c4r1h"
Prometheus → Scrapes → Node Exporter
```

This happens automatically based on the configured scrape interval.

Default:

```text id="qkmg8t"
15 seconds
```

---

# **11. Extending Node Exporter**

Node Exporter supports:

# **Pluggable Metric Collectors**

This means you can extend it to collect additional metrics beyond the default set.

Examples:

* Hardware sensors
* RAID controllers
* Systemd services
* Custom collectors

---

# **12. Why Node Exporter Is Important**

Node Exporter is one of the most widely used exporters because it provides:

* Infrastructure monitoring
* Server health visibility
* Performance insights
* Capacity monitoring

It is commonly used with:

* Grafana dashboards
* Alertmanager
* Kubernetes monitoring
* Cloud infrastructure

---

# **13. Typical Monitoring Stack**

A common monitoring architecture:

```text id="n22lwr"
Linux Server
    ↓
Node Exporter
    ↓
Prometheus
    ↓
Grafana
```

Where:

* Node Exporter collects metrics
* Prometheus stores metrics
* Grafana visualizes metrics

---

# **14. Key Concepts**

## Node

A Linux/Unix server or machine.

---

## Node Exporter

An exporter that exposes Linux system metrics.

---

## Scraping

Prometheus periodically pulling metrics from Node Exporter.

---

# **15. Summary**

Node Exporter is:

* The official Prometheus exporter for Linux/Unix systems
* Used for infrastructure and system monitoring
* Exposes metrics through HTTP
* Scraped by Prometheus on regular intervals

---

# **Key Takeaways**

* Node Exporter is unrelated to Node.js
* It collects Linux system metrics
* Metrics are exposed on port `9100`
* Prometheus scrapes metrics from Node Exporter
* It is part of the official Prometheus ecosystem
* Additional collectors can extend its functionality
